# 语言系统需求规格

## 行为概述

PlainTab 支持 16 种界面语言。用户切换语言后，页面上所有文字——标题、按钮、占位符、提示——全部即时切换，无需刷新。

语言选择不仅影响界面文字，还会影响 Bing 壁纸的来源地区。例如选择日语则从日本 Bing 获取壁纸，选择德语则从德国 Bing 获取。

### 语言检测（首次使用）

用户首次打开新标签页（未手动选过语言）时，系统自动检测浏览器的语言：

1. 扩展模式 → 使用浏览器的界面语言
2. 网页模式 → 使用浏览器的首选语言
3. 都获取不到 → 默认英语

检测采用两级匹配：
- **精确匹配**：浏览器说 `zh-CN`，恰好有繁体中文 → 直接命中
- **前缀匹配**：浏览器说 `zh-HK`，没有精确匹配，取前缀 `zh`，找到第一个以 `zh` 开头的语言 → 用简体中文
- **两级都失败** → 回退英语

若用户手动选择过语言，之后直接使用用户的选择，不再自动检测。

### 语言切换

点击右上角地球按钮弹出语言面板。面板内列出 16 种语言，当前选中的高亮加粗。点击某个语言：

1. 保存语言选择
2. 即时刷新页面上所有文字
3. 更新语言面板的高亮
4. 关闭语言面板

整个切换是即时的，零延迟。

切换语言后以下所有元素的文字都会刷新：页面标题、搜索框占位符、引擎图标提示、语言按钮提示、设置按钮提示、设置面板内的所有文字、一级面板的模式标签和描述、模态窗口内的所有文字、命令面板内的按钮和提示（如果打开的话）、语言面板自身。

## 约束清单

### 语言检测

- 用户手动选过语言后不再自动检测，始终使用用户选择
- `pt`（葡萄牙语）的前缀是 `pt`，所以 `pt-BR` 浏览器的前缀匹配会命中葡萄牙语
- 存储的语言值损坏或无效 → 重新自动检测
- 翻译表加载失败 → 仅使用浏览器原生翻译兜底

### 翻译回退链

查找一段文字的翻译时，按以下优先级：

1. 浏览器原生翻译（仅扩展模式下可用，覆盖约 2 个基础 key）
2. PlainTab 自己维护的翻译表中当前语言的翻译
3. 翻译表中英语版本（英语是兜底语言）
4. 若英语也没有 → 显示 key 本身（开发调试信号）

### 切换行为

- 点击已是当前的语言 → 不做任何事
- 切换后所有可见文字即时刷新
- 语言面板首次打开时创建按钮，之后打开只更新文字和高亮，不重建 DOM
- 如果模态窗口或命令面板正打开，切换语言后其中的文字也会同步刷新

### 新增语言

要支持一种新语言，只需添加一份翻译表（约 45 个翻译条目）和一个语言列表条目，无需改任何其他代码或界面。
