# 语言系统需求规格

## 一、概述

PlainTab 支持 16 种语言的界面翻译。用户切换语言后，页面上所有的文字——标题、按钮、占位符、提示——全部即时切换，不需要刷新页面。

这套系统只在「页面 UI 层」生效。Chrome 扩展的元数据（扩展名、描述）走 Chrome 自己的 `_locales/` 翻译机制，是另一条独立的轨道。

## 三、语言检测

### 3.1 首次使用（没选过语言）

系统自动检测用户浏览器的语言：

1. 扩展模式 → 用 Chrome 原生 API `chrome.i18n.getUILanguage()`，返回浏览器界面语言
2. 网页模式 → 用 `navigator.language`，返回浏览器首选语言
3. 都拿不到 → 默认英文

### 3.2 两级匹配

拿到的浏览器语言代码不一定精确匹配 PlainTab 的 16 种语言，所以做两级匹配：

- **第一级——精确匹配**：比如浏览器说 `zh-CN`，PlainTab 恰好有 `zh-CN` → 直接命中
- **第二级——前缀匹配**：比如浏览器说 `zh-HK`，PlainTab 没有 `zh-HK`，但取前缀 `zh`，找到第一个以 `zh` 开头的语言 `zh-CN` → 用 `zh-CN`
- **两级都失败**：回退到 `en`（英文）

特别注意：`pt`（葡萄牙语）的前缀就是 `pt`，所以 `pt-BR` 的浏览器会匹配到 `pt`。

### 3.3 用户选过语言之后

- 用户选择的语言存到 `ptab_lang`
- 下次打开新标签页，**直接读 `ptab_lang`**，不再检测浏览器语言
- 只有 `ptab_lang` 不存在或损坏时，才重新检测

## 四、翻译查找机制

整个页面只有一个翻译入口：`t(key)`。查找优先级从高到低：

```
第 1 优先：chrome.i18n.getMessage(key)    ← 仅扩展模式，从 _locales/ 翻译文件里找
第 2 优先：I18N[当前语言][key]            ← 自己的翻译表，16 种语言全覆盖
第 3 优先：I18N['en'][key]               ← 英语兜底
第 4 优先：key 本身                       ← 显示原始 key，开发者调试信号
```

- 没有参数替换功能——翻译字符串就是纯文本
- 一般用于：`元素.textContent = t('key')`、`元素.placeholder = t('key')`、`元素.title = t('key')`

## 六、语言切换面板

### 6.1 UI

- 从右上角地球按钮打开
- 磨砂玻璃面板，位置在语言按钮正下方
- 列出 16 种语言，当前语言高亮加粗
- 点击某个语言 → 立刻切换

### 6.2 切换流程

1. 用户点击某个语言
2. 如果已经是当前语言 → 不做事
3. 如果不同：
   - 保存到 `localStorage.ptab_lang`
   - 更新内存中的 `currentLang`
   - 刷新页面上**所有**用了 `t()` 的文字（逐个 DOM 元素更新）
   - 更新语言面板里的高亮
4. 关闭语言面板

整个切换过程是**同步的**——用户点击后立刻生效，零延迟。

### 6.3 首次创建 vs 后续更新

- 第一次打开语言面板：创建 16 个按钮 DOM，绑定点击事件
- 之后每次打开：只更新按钮上的文字和高亮状态，不重建 DOM
- 这样反复开关语言面板不会产生性能开销

### 6.4 哪些 DOM 会被刷新

切换语言时，以下所有元素的文字都会更新：
- 页面标题、搜索框占位符、引擎图标提示
- 语言按钮提示、设置按钮提示
- 设置面板标题、壁纸信息行
- 上传按钮、重置按钮、高级选项按钮
- 搜索设置标签（3 个）、下拉框选项（3 个）、恢复默认按钮
- 本地图库文字
- 语言面板自身（标题 + 16 个按钮文字 + 高亮）
- 如果命令面板开着 → 命令栏按钮文字也刷新

## 八、各种情况处理

| 情况 | 处理 |
|------|------|
| 浏览器语言不在 16 种中 | 前缀匹配 → 匹配不到就英文 |
| `ptab_lang` 存了无效值 | 忽略，重新检测浏览器语言 |
| 语言面板打开时切换语言 | 切换后关闭面板 |
| 翻译表缺某个 key | 回退英语 → 英语也没有就显示 key 本身 |
| `window.I18N` 没加载（`languages.js` 加载失败） | `I18N` 设为空对象 `{}`，只用 Chrome API 兜底 |
| 网页模式没有 `chrome.i18n` | 直接走 I18N 表，第一步跳过 |
| 用户选的语言和浏览器语言不同 | 用户选择优先——存了 `ptab_lang` 就不再检测 |

## 十、添加新语言

要支持一种新语言，只需要做三件事：
1. 在 `js/languages.js` 的 `I18N` 对象里加一个新语言的完整翻译（全部 27 个 key）
2. 在 `LanguageList` 数组里加 `{code, name}` 条目
3. 可选的：在 `_locales/` 下创建对应目录和 `messages.json`（仅扩展名和描述两个 key）

不用改任何其他代码。
