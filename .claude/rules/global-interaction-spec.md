# 全局交互需求规格

## 一、概述

全局交互系统是 PlainTab 的「神经系统」——它管理所有键盘快捷键、鼠标行为、面板之间的协调、页面启动流程。它不负责具体的功能实现，而是负责在正确的时间把正确的操作路由到正确的模块。

---

## 二、页面启动流程

每次打开新标签页，整个页面从零开始。启动顺序决定了用户体验的好坏。

### 2.1 文件加载顺序（严格、不可变）

```
1. HTML 解析 → #wallpaperBack 出现
2. preload.js 同步执行（立即写缩略图到背景）
3. #wallpaperFront 出现
4. 其他 DOM 元素（搜索栏、按钮、面板、命令面板等）
5. languages.js 加载（设置 window.I18N + window.LanguageList）
6. newtab.js 加载（主程序 IIFE 立即执行）
```

- preload.js 必须在 `#wallpaperBack` 之后但在浏览器画第一帧之前执行
- 如果 preload.js 在 `#wallpaperBack` 之前 → 找不到 DOM 元素 → 白屏
- languages.js 必须在 newtab.js 之前 → 否则翻译表不存在

### 2.2 主程序启动步骤

1. **检测环境**：判断是扩展模式还是网页模式
2. **检测语言**：读 `ptab_lang`，没有就自动检测浏览器语言
3. **恢复设置**：从 localStorage 读取搜索模式、透明度、引擎、快捷键等
4. **加载壁纸**：这是最重的步骤——本地轮换 → Bing 缓存 → Bing 网络
5. **扩展模式适配**：如果检测到是扩展 → 隐藏引擎选择器、绑定 Chrome 搜索 API
6. **绑定所有事件**：键盘、鼠标、点击、窗口大小变化等
7. **页面就绪**

## 三、键盘快捷键

所有快捷键通过一个全局 keydown 事件监听器处理，按优先级路由：

### 3.1 优先级链

```
如果命令面板开着：
    → 所有键盘事件先交给命令面板处理（导航、回车、Escape 返回等）
    → 命令面板不处理的才往下传递

如果命令面板没开：
    Escape       → 关闭所有面板（设置、语言）→ 隐藏角落按钮
    Ctrl+K       → 打开 Normal 命令面板
    Ctrl+Shift+K → 打开 Hidden 命令面板
    Ctrl+Shift+W → 切换设置面板
    Enter        → 如果搜索框有焦点 → 执行搜索
```

### 3.2 互斥规则

- Normal 面板开着的时候按 `Ctrl+Shift+K` → 不切换，面板内显示提示「当前普通模式 — 请按 Esc 关闭后再试」，2 秒后自动消失
- Hidden 面板开着的时候按 `Ctrl+K` → 不切换，面板内显示提示「当前隐藏模式 — 请按 Esc 关闭后再试」，2 秒后自动消失

### 3.3 快捷键配置

- Normal 面板快捷键 → 设置面板里可配，存 `ptab_shortcut_hotkey`
- Hidden 面板快捷键 → 命令面板内部可配，存 `ptab_shortcut_hidden_hotkey`
- 默认值：Normal = `Ctrl+K`，Hidden = `Ctrl+Shift+K`

## 四、鼠标行为

### 4.1 全局鼠标追踪

页面持续追踪鼠标位置（用 requestAnimationFrame 节流，不卡 UI）：

### 4.2 右上角热区 → 角落按钮

- 热点区域：右上角 180×130 像素范围（`x > 窗口宽度 - 180 && y < 130`）
- 鼠标进入 → 立即显示齿轮按钮和语言按钮
- 鼠标离开 → 400ms 延迟后隐藏（短延迟防止鼠标快速路过时闪烁）
- **面板打开时不隐藏**——因为用户需要点击按钮来关闭面板
- 语言按钮默认完全隐藏（opacity:0），齿轮按钮默认半透明（opacity:0.45）

### 4.3 屏幕中心区域 → 搜索栏

- 中心区域：宽度 30%-70%，高度 42%-58%
- 仅当搜索模式为「悬停显示」时生效
- 鼠标进入 → 搜索栏立即出现
- 鼠标离开 → 150ms 后消失
- 在搜索框里打字时 → 无论如何不消失

### 4.4 点击面板之外 → 关闭面板

- 全局 click 监听器
- 点击命令面板遮罩（半透明黑色背景）→ 关闭命令面板
- 点击设置面板之外的区域（且点的不是齿轮按钮）→ 关闭设置面板
- 点击语言面板之外的区域（且点的不是语言按钮）→ 关闭语言面板
- 点击任何空白区域 → 搜索框获得焦点

## 五、面板协调

### 5.1 面板互斥规则

- 设置面板 ↔ 语言面板：互斥，打开一个关闭另一个
- 命令面板 ↔ 设置面板 / 语言面板：不是互斥——命令面板是覆盖层（全屏遮罩），打开时设置/语言面板自动被遮住
- 同一个面板不能重复打开（幂等）

### 5.2 面板状态管理

每个面板用一个布尔变量追踪状态：
- `isSettingsPanelOpen` — 设置面板是否开着
- `isLangPanelOpen` — 语言面板是否开着
- `isPaletteOpen` — 命令面板是否开着

关闭函数是幂等的——已经关了再调用也不报错。

### 5.3 面板打开的互斥行为

- 打开命令面板 → 不主动关设置/语言面板（命令面板覆盖在上面）
- 关闭命令面板 → 保持设置/语言面板的原始状态不变

### 5.4 面板关闭时角落按钮行为

- 关闭所有面板后 → 根据鼠标是否在热区决定角落按钮显隐
- 如果鼠标在热区 → 保持显示
- 如果鼠标不在热区 → 400ms 后隐藏

## 六、角落按钮交互

### 6.1 齿轮按钮（#settingsBtn）

- 点击 → 打开/关闭设置面板
- 面板关闭时：半透明（opacity 由设置决定）
- 面板打开时：完全不透明 + 旋转 60 度
- 鼠标悬停：完全不透明 + 旋转 30 度 + 出现阴影
- 鼠标按下：旋转 30 度 + 缩小到 92%

### 6.2 语言按钮（#langBtn）

- 点击 → 打开/关闭语言面板
- 默认完全隐藏（opacity:0）
- 鼠标在热区时显示
- 悬停：略微放大（1.05 倍）
- 按下：缩小到 92%

---

## 七、窗口大小变化

- 响应式断点：480px
- 窗口小于 480px → 按钮缩小、面板缩小、搜索栏变宽
- 热区检测用的是百分比，自动适应窗口大小变化

---

## 八、各种情况处理

| 情况 | 处理 |
|------|------|
| 快速连续按 Ctrl+K | 只打开一次——命令面板已打开后不再重复打开 |
| 命令面板动画还没播完就按 Esc | 关闭动画覆盖打开动画 |
| 同时按 Ctrl+K 和 Ctrl+Shift+K | 只会触发一个——键盘事件是串行的 |
| 在 iframe 里点击 | 全局 click 不处理（只监听 document） |
| 多个面板同时打开（极端情况） | 理论上不会发生——每个打开函数都有状态检查 |
| 点击面板上的空白区域 | 被 `stopPropagation` 拦截，不会触发「点击面板外关闭」 |
| 面板打开时按 Ctrl+Shift+W | 关闭设置面板 |
| 窗口失去焦点 | 不处理——键盘事件需要焦点在页面上 |

---

## 九、与其它模块的关系

全局交互系统是顶层的协调者，它依赖所有模块：

| 依赖模块 | 怎么依赖 |
|---------|---------|
| 命令面板 | 调用打开/关闭/键盘导航 |
| 设置面板 | 调用打开/关闭 |
| 语言面板 | 调用打开/关闭 |
| 搜索栏 | 触发搜索执行、控制显示/隐藏、管理焦点 |
| 壁纸系统 | 启动时触发壁纸加载 |

| 被谁依赖 | 怎么依赖 |
|---------|---------|
| 无 | 全局交互是顶层，不被其它模块调用 |
