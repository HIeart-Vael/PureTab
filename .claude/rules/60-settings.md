# 设置面板需求规格

## 行为概述

设置系统分两层：

- **一级面板**：右上角齿轮打开，展示当前壁纸来源、画廊预览、上传入口、进入完整设置的按钮和 GitHub/版本信息。
- **模态窗口**：完整设置中心，包含“界面 / 壁纸 / 快捷键 / 数据 / 关于”五个分页。

`settings-bootstrap.js` 负责轻量首屏设置与一级入口；`settings-panel.js` 是完整设置模块，按需懒加载。

### 一级面板

一级面板内容：

- 壁纸来源 chip：Bing、Upload、Folder、RSS、API 使用不同颜色。
- 画廊锚点：按当前来源渲染缩略图。
- `+` 上传按钮：用于本地上传图片，最多 12 张。
- 底部栏：高级设置按钮、GitHub 链接、版本文本。

画廊行为：

| 来源 | 画廊 |
|------|------|
| Bing | 单张当前/preview 缩略图，不可删除 |
| API | 单张 `api` 缩略图，不可删除 |
| Upload | 最多 12 张，支持删除、拖拽排序和继续添加 |
| Folder | 当前图和少量随机袋预览，不展示全量文件夹 |
| RSS | 当前 active RSS source 的缓存图，不可在画廊删除 |

一级面板的 `+` 始终是上传入口：它会懒加载完整设置模块并调用 `pickUpload()`，不会修改壁纸配置草稿。

### 模态窗口

模态窗口左侧是 tab，右侧是内容区。打开方式：

- Ctrl/Cmd+Shift+W。
- 一级面板“高级设置”按钮。

关闭方式：

- Escape。
- 点击遮罩区域。
- 再按 Ctrl/Cmd+Shift+W。

点击窗口内部不会关闭。完整设置打开时一级面板关闭。

## 界面分页

界面分页的设置即时保存，无需 `应用配置`：

- 搜索栏显示、位置、宽度、圆角、背景、模糊、图标位置。
- 搜索历史数量：关闭 / 5 条 / 10 条。
- 搜索引擎（仅网页模式显示）。
- 壁纸适配、焦点、背景模糊、遮罩。
- 壁纸主题色。
- 图标透明度、面板透明度、界面圆角。

“恢复默认设置”只恢复界面偏好：

- 搜索栏始终显示、居中、胶囊、宽 560。
- 搜索框背景 0.1、模糊 24。
- 搜索历史恢复为 5 条。
- 壁纸 cover/center、无遮罩、无背景模糊、主题色关闭。
- 图标透明度 0.45、面板透明度 0.88、界面圆角 soft。
- 网页模式恢复搜索引擎为 Google；扩展模式不显示引擎行。

## 壁纸分页

壁纸分页采用固定头部 + 可滚动内容 + 固定底部应用栏：

- `wallpaper-tab-shell` 使用三行 grid。
- `wallpaper-tab-body` 独立滚动。
- 顶部标题和底部 `应用配置` 不随内容滚动。

### 草稿与应用

打开壁纸分页时，从当前 `WallpaperData.loadWallpaper()` 克隆草稿。用户修改来源或配置时只改草稿：

- 关闭模态窗口：丢弃草稿。
- 点击来源 drawer：只切换草稿 activeSource 和展开区域。
- 点击 `应用配置`：通过校验后写入存储并触发 `window.reloadWallpaper()`。

`应用配置` 按钮启用条件：

| 来源 | 条件 |
|------|------|
| Bing | 草稿有变更 |
| Upload | 草稿有变更 |
| Folder | 草稿有变更、浏览器支持、且本次已选择并准备好有效文件夹 |
| RSS | 草稿有变更、已选择 RSS source、该 source 测试为绿色 |
| API | 草稿有变更、已选择当前 apiType 下的 source、该 source 测试为绿色 |

如果没有有效更改，按钮禁用并显示“没有未应用更改”。

### 来源切换清缓存确认

应用时比较保存值和草稿：

- 旧来源是 Bing：不确认，不清 Bing 缓存。
- 旧来源不是 Bing 且存在缓存：先确认“切换来源会丢弃当前来源已缓存的壁纸数据”。
- 用户取消确认：不保存、不清理、不 reload。
- 用户确认：先清旧来源缓存，再应用新配置。

配置始终保留；只清理旧来源大文件、缩略图、meta 和运行态缓存。

### Bing 配置区

Bing 无额外配置。说明当前语言会决定后续 Bing 市场。

### Upload 配置区

只提示用户到一级面板点击 `+` 上传。选择 Upload 后点击 `应用配置` 不弹文件选择器。

### Folder 配置区

控件：

- 当前文件夹标签。
- `选择文件夹` 按钮。
- 只读轮换方式：随机。
- 状态/错误提示。

点击 `选择文件夹`：

1. 调用 `WallpaperFolder.pickDirectory()`。
2. 调用 `prepareMount()` 扫描首批、读取首张可用图并生成 preview。
3. 成功后只更新草稿和 `wallpaperDraftFolderMount`。
4. 失败或取消时不改已保存配置。

应用成功后才保存目录句柄、文件索引、folder 配置和状态。

### RSS 配置区

控件：

- RSS source 列表，最多 5 个。
- 每行单选、名称、URL、测试按钮、删除按钮、红/绿状态点。
- 新增行：名称 + URL + 添加。
- 自动拉取间隔：关闭 / 1 天 / 3 天 / 7 天。
- 摘要位置、摘要展示方式、显示摘要、显示正文链接。
- 状态/测试提示。

新增 source 初始为红点；测试通过后变绿。URL 必须是 HTTPS。删除最后一个 RSS source 时恢复默认内置 sources。

### API 配置区

控件：

- 横向互斥 tab：图片直链/重定向、JSON。
- 每个 tab 内独立 source 列表，最多 5 个。
- 每行单选、名称、URL、测试按钮、删除按钮、红/绿状态点。
- 新增行：名称 + URL；JSON 模式额外有 JSON 图片路径。
- 自动拉取间隔：关闭 / 每次打开 / 1 天 / 3 天 / 7 天。

新增 source 初始为红点；当前选择的 source 必须测试通过才能应用。API 测试通过后，应用配置优先复用测试时下载到的图片 Blob。

### 恢复默认壁纸设置

壁纸分页底部有危险按钮。确认后：

- activeSource 切回 Bing。
- 保留 Bing Blob/缩略图/meta/preview。
- 清理 Upload、Folder、RSS、API 的缓存数据。
- 重置 provider 配置为默认值。
- reload 当前壁纸。

## 快捷键分页

设置项：

- 普通命令面板快捷键，默认 `ctrl+k`。
- 隐藏命令面板快捷键，默认 `ctrl+shift+k`。
- 显示推荐开关。

## 数据分页

数据分页用于导入/导出所有用户配置：

- 明文 JSON 导出：文件名为 `plaintab-config-YYYY-MM-DD.json`，便于留存、排查和人工查看。
- 加密文件导出：文件名为 `plaintab-config-YYYY-MM-DD.ptab`，用户输入口令后导出；实现上先压缩 JSON，再用 Web Crypto PBKDF2-SHA256 + AES-GCM 加密。
- 导入：支持 PlainTab JSON 备份和加密备份；加密备份需要输入同一口令。

导入覆盖语言、界面偏好、壁纸 localStorage 模型、缩略图/预览、命令面板普通与隐藏列表、最近访问、快捷键、推荐开关、视图模式和图标缓存。导入成功后重新加载设置状态、刷新画廊、刷新已加载的命令面板缓存，并触发壁纸 reload。

备份不导出 IndexedDB 中的原图 Blob 或文件夹句柄，因此跨设备导入后本地上传原图和文件夹授权需要用户重新提供；Bing 兜底仍由运行时恢复。

录制规则：

- 必须包含 Ctrl/Cmd。
- 必须有且仅有一个 A-Z 字母键。
- 可包含 Shift。
- 两个快捷键不能相同。
- Escape 或点击外部取消录制。

快捷键设置保存到 `ptab_shortcuts.settings`。

## 关于分页

展示 PlainTab 名称、`v3.2.0` 文本、简介、GitHub 链接和 MIT License。该版本文本是当前 UI 显示值；发布前如同步 manifest 版本，需要同时检查这里和一级面板版本文本。

## 约束清单

- 外观设置即时保存；壁纸来源/配置必须走草稿 + `应用配置`。
- 壁纸 tab 的头部和应用栏不能随滚动内容移动。
- RSS/API source 的红绿点由 `test.status` 和字段 hash 共同决定，字段变更后旧测试不能继续算通过。
- 文件夹应用必须有本次有效 mount；只显示旧路径标签不能直接重新保存为新文件夹授权。
- 面板关闭时释放画廊生成的临时 Blob URL。
- 删除上传图遵循先断引用、再删缩略图/meta、最后删 IDB Blob。
- 全局配置导入/导出只在“数据”分页；命令面板内的 `/import` / `/export` 仍只处理快捷链接/书签边界。
