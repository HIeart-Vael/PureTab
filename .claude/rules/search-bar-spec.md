# 搜索栏需求规格

## 一、概述

搜索栏是新标签页的核心交互入口——用户在这里输入关键词、按回车搜索。它位于屏幕正中央，半透明磨砂玻璃风格浮动在壁纸之上。

搜索栏在不同环境下的行为不一样：Chrome 扩展版本必须用浏览器内置搜索引擎（Chrome 商店政策要求），网页版本可以自由切换 4 种搜索引擎。

---

## 二、搜索栏的三种显示模式

| 模式 | 行为 | 默认？ |
|------|------|--------|
| 始终显示（always） | 搜索栏永远可见，不用做任何操作 | ✅ |
| 悬停显示（hover） | 鼠标移到屏幕中间时才出现，移开消失 | |
| 始终隐藏（never） | 搜索栏永远不显示 | |

- 用户在设置面板里选择模式
- 模式持久化到 `ptab_search_mode`
- **hover 模式细节：**
  - 鼠标移到屏幕中心区域（宽度 30%-70%，高度 42%-58%）→ 搜索栏立即出现
  - 鼠标离开中心区域 → 150ms 后消失（短延迟防止误触消失）
  - 如果正在搜索框里打字 → 无论如何都不消失
- **always 模式细节：** 设置面板里切换到此模式时，搜索栏立即出现
- **never 模式细节：** 设置面板里切换到此模式时，搜索栏立即消失

---

## 三、搜索引擎切换

### 3.1 扩展模式（Chrome/Edge 商店版）

- 只能用浏览器自带的默认搜索引擎
- 搜索引擎图标显示为放大镜（静态，不能点）
- 设置面板里「搜索引擎」这一行隐藏
- 按回车时调用 `chrome.search.query()` API 在当前标签页搜索

### 3.2 网页模式（自己部署的网页版）

支持 4 种搜索引擎：

| 引擎 | 搜索 URL 格式 |
|------|-------------|
| Google | `https://www.google.com/search?q=关键词` |
| Bing | `https://www.bing.com/search?q=关键词` |
| Baidu | `https://www.baidu.com/s?wd=关键词` |
| DuckDuckGo | `https://duckduckgo.com/?q=关键词` |

- 点击搜索引擎图标 → 切换到下一个引擎（循环）
- 设置面板里有下拉框可以选引擎
- 引擎选择持久化到 `ptab_search_engine`
- 图标是内嵌的 SVG，不需要外部图片文件
- 图标颜色受透明度设置影响

## 四、搜索栏的 UI 结构

```
┌──────────────────────────────────────┐
│      [输入关键词…]       🔍 引擎图标 │
└──────────────────────────────────────┘
```

- 整体是胶囊形（圆角 28px），居中浮动
- 背景半透明白色磨砂玻璃
- 输入框背景透明，字号 18px，白色文字
- 桌面宽度 560px，小屏幕（480px 以下）宽度 90vw
- 引擎图标 40×40px，鼠标悬停有缩放效果（仅网页模式）

### 占位符文字

占位符随当前语言变化，比如中文显示「在 Web 中搜索…」，英文显示「Search the Web…」。

### 图标提示

鼠标悬停在引擎图标上时，显示当前搜索引擎名称（如「Google」）+ 操作提示（如「点击切换 | 回车搜索」），扩展模式下只显示「回车搜索」。

---

## 五、搜索执行

### 回车搜索流程

1. 用户输入关键词，按回车
2. 去除关键词首尾空格
3. 关键词为空 → 不执行搜索
4. 扩展模式 → 调用 `chrome.search.query({text: 关键词, disposition: 'CURRENT_TAB'})`，在当前标签页搜索
5. 网页模式 → `window.open(搜索引擎URL + 关键词, '_self')`，当前页面跳转搜索

### 聚焦搜索栏

- 打开新标签页后，点击页面空白区域 → 搜索栏自动获得焦点
- 如有命令面板等浮层打开，点击浮层内的空白区域不会触发聚焦

---

## 六、与设置面板的联动

设置面板里的高级选项区有三个跟搜索相关的控件：

| 控件 | 作用 | 影响 |
|------|------|------|
| 搜索栏显示（下拉框） | 切换 always/hover/never | 立即改变搜索栏可见性 |
| 图标不透明度（滑块+数字） | 调整 0~1 | 立即改变所有图标透明度 |
| 搜索引擎（下拉框） | 选 Google/Bing/Baidu/DDG | 立即改变引擎图标和搜索目标 |

- 控件改动即时保存，无需点确认
- 改搜索引擎下拉框时，引擎图标同步切换
- 改透明度时，用一个 CSS 变量控制所有图标（`--icon-opacity`），一处改处处生效
- 扩展模式下「搜索引擎」这一行隐藏，下拉框不显示

---

## 七、扩展模式判定

系统启动时自动检测当前环境：

```
if (Chrome 扩展 API 可用 && 有扩展 ID) → 扩展模式
else → 网页模式
```

- 有扩展 ID 说明是通过 `chrome://extensions` 加载的正式扩展
- 网页版可能是部署在 Cloudflare Workers 或 GitHub Pages 上的版本

---

## 八、各种情况处理

| 情况 | 处理 |
|------|------|
| 搜索关键词为空 | 不搜索 |
| 扩展模式下用户选了搜索引擎 | 不影响，扩展模式强制用浏览器默认引擎 |
| 从未设置过搜索引擎 | 默认 Google |
| localStorage 里引擎值损坏 | 回退到 Google |
| 从未设置过显示模式 | 默认 always |
| 输入框正在聚焦中，鼠标移出 | hover 模式下不隐藏搜索栏 |
| 命令面板打开时鼠标移到中心 | 搜索栏不弹出（被命令面板遮罩覆盖） |
| 小屏幕（手机/窄窗口） | 搜索栏变宽（90vw），占满屏幕宽度，确保可用 |

---

## 九、与其它模块的关系

| 依赖谁 | 依赖什么 |
|--------|---------|
| 数据存储 | 持久化搜索模式、引擎、透明度 |
| 语言系统 | 搜索栏占位符文字、引擎图标提示文字 |
| 设置面板 | 搜索相关的设置控件 |

| 被谁依赖 | 怎么依赖 |
|---------|---------|
| 全局交互 | 回车键触发搜索；鼠标移到中心触发显示/隐藏 |
| 应用启动 | 启动时恢复搜索设置 |
