# 本地图库需求规格

## 一、概述

本地图库负责管理用户上传的壁纸图片。当壁纸模式切换到「本地轮换」后，设置面板内会显示一个 3 列 × 4 行的缩略图网格，用户可以在这里浏览、添加、删除和拖拽排序自己的壁纸。

图库的 UI 渲染在设置面板内部——打开设置面板时，如果当前是本地模式，面板里就会显示图库。

---

## 二、数据模型

每条本地图片存三份数据：

| 存在哪 | Key | 内容 |
|--------|-----|------|
| localStorage | `ptab_img_order` | 数组，图片 ID 的列表，顺序决定轮换顺序，是唯一真源 |
| localStorage | `ptab_img_thumbs` | 对象，`{id: "缩略图 base64 字符串"}` |
| localStorage | `ptab_img_meta` | 对象，`{id: {name: 文件名, size: 文件大小}}` |
| IndexedDB | `ptab_img_<id>` | `{blob: 图片文件本体, mime: 文件类型, name: 文件名}` |

- 图片 ID 是 36 进制编码的时间戳 + 随机后缀，保证不重复
- **图片数量上限：12 张**。满了就不能再加

---

## 三、图库 UI

### 3.1 什么时候显示

- 设置面板打开 + 当前是本地模式 → 显示图库
- 设置面板打开 + 当前是 Bing 模式 → 不显示图库，只显示上传和重置按钮
- 模式切换（比如删除最后一张后自动切回 Bing）→ 图库立即消失

### 3.2 网格布局

- 3 列 × N 行，N 取决于图片数量（最多 4 行，12 张）
- 每个格子：正方形缩略图 + 右上角红色删除按钮（✕）
- 格子之间有间距
- 不足一行有空位

### 3.3 缩略图显示

- 优先用 `ptab_img_thumbs` 里的缓存缩略图（base64，零 IO）
- 如果缓存缺失 → 从 IndexedDB 读取原图 → 生成缩略图 → 补写缓存
- 如果原图也丢了 → 格子显示为空，不影响其他图片
- 缩略图显示期间生成的 Blob URL 需要在图库关闭时释放——否则内存泄漏

### 3.4 右上角删除按钮

- 每张缩略图右上角有一个红色的 ✕
- 点击 → 删除这张图片
- 删除顺序（见第四节）

### 3.5 底部的「＋」按钮

- 图库最后一个位置总是一个「＋」按钮
- 点击 → 弹出文件选择器 → 选一张图片上传
- 如果已满 12 张 → 「＋」按钮不显示

---

## 四、上传图片

### 4.1 单张上传

- 用户在文件选择器里选一张图片
- 系统：
  1. 生成唯一 ID
  2. 创建缩略图（640px 宽 JPEG）
  3. 原图 blob 存到 IndexedDB
  4. ID 追加到 `ptab_img_order`
  5. 缩略图存入 `ptab_img_thumbs`
  6. 元数据（文件名、大小）存入 `ptab_img_meta`
  7. 设置壁纸模式为「本地」
  8. 刷新图库显示
  9. 如果是第一张 → 立即展示为当前壁纸

### 4.2 批量上传

- 文件选择器可以多选（按住 Ctrl/Shift）
- 批量上传时一次一个处理（串行），不是并行
- 第一张成功 → 展示为壁纸
- 其余的静默保存到图库
- 计算公式：剩余空位 = 12 - 已有数量，超过的不处理

### 4.3 上传的崩溃保护

写入顺序很重要——如果中途崩溃，不能出现数据不一致：

```
正确顺序：
  1. 原图 blob → IndexedDB（重数据先落盘）
  2. ID 加入 order → 缩略图写入 thumbs → 元数据写入 meta（轻数据后关联）
```

- 如果在第 1 步后崩溃：blob 存在 IDB 但 order 里没有 → 孤儿 blob，安全忽略
- 在第 2 步中崩溃：order 和 thumbs 可能不一致 → 下次加载时发现缺失缩略图会自愈重新生成

---

## 五、删除图片

### 5.1 删除顺序（同样考虑崩溃保护）

```
正确顺序：
  1. 从 order 数组里移除这个 ID（先断引用——图片立刻不可达）
  2. 从 thumbs 里删掉缩略图
  3. 从 meta 里删掉元数据
  4. 从 IndexedDB 删掉 blob（最后删重数据）
```

- 如果在第 1 步后崩溃：图片不可达，但 blob 还在 IDB → 无害孤儿
- 如果在第 4 步后崩溃：完全清理干净

### 5.2 删除最后一张

- 删完后 order 为空 → 自动切换回 Bing 模式
- 清理 `ptab_local_index` 和 `ptab_preview_thumb`
- 设置面板里的图库消失，恢复为「上传 + 重置」两个按钮

---

## 六、拖拽排序

### 6.1 触发方式

- 在缩略图上长按（手机）或按住鼠标拖拽（桌面）→ 进入拖拽模式
- 拖拽中的缩略图放大、加阴影，跟随鼠标/手指移动
- 拖到两个格子之间 → 其他格子自动让位（动画过渡）
- 松开 → 图片插入新位置，更新 `ptab_img_order`

### 6.2 拖拽中的视觉反馈

- 被拖拽的缩略图：略微放大 + 深色阴影 + 跟随光标
- 目标位置：显示一个脉动的占位框（边框闪烁动画）
- 其他格子：平滑移动让位

### 6.3 排序后的持久化

- 拖拽结束 → 更新 `ptab_img_order` 数组 → 保存到 localStorage
- 轮换顺序立即生效（下次打开新标签页按新顺序播）

---

## 七、重置（切回 Bing）

- 用户在设置面板点「恢复 Bing 每日壁纸」→ 确认 → 执行重置
- 重置步骤：
  1. 把所有 `ptab_img_<id>` 从 IndexedDB 批量删除（一次事务完成）
  2. 清空 localStorage 中的 order、thumbs、meta、local_index、preview_thumb
  3. 切换 `ptab_mode` 为 `'bing'`
  4. 刷新设置面板（图库消失，恢复上传/重置按钮）
  5. 壁纸引擎加载 Bing 壁纸
- 如果中途崩溃：IDB 已清空但 order 还有引用 → 下次加载发现 blob 缺失，跳过该图片
- **不可撤销**——确认对话框里明确告知用户

---

## 八、Blob URL 内存管理

- 图库渲染时，如果某张图片的缩略图缓存缺失，需要从 IndexedDB 读取 blob → 生成临时 Blob URL → 用于显示
- 所有生成的临时 Blob URL 必须记下来
- 图库关闭（设置面板关闭或切换到 Bing 模式）→ 逐个 `URL.revokeObjectURL()` 释放
- 如果不释放：每次打开设置面板都创建新 URL，内存只增不减

---

## 九、各种情况处理

| 情况 | 处理 |
|------|------|
| 上传时文件选择器取消 | 什么都不做 |
| 上传的图片损坏/不是图片 | 静默跳过，不影响已有图片 |
| 上传已满 12 张 | 「＋」按钮不显示，文件选择器不再能打开 |
| 拖拽中途松开在外面 | 图片回到原位，不保存更改 |
| 删除最后一张 | 自动切回 Bing 模式 |
| IndexedDB 读取 blob 失败 | 该格子显示为空，不报错 |
| localStorage 写入缩略图失败（配额满） | 缩略图不存，下次重新生成 |
| JSON 解析数据损坏 | 回退到默认空值（`[]` / `{}`） |

## 十、与其它模块的关系

| 依赖谁 | 依赖什么 |
|--------|---------|
| 壁纸系统 | 上传触发壁纸展示、删除触发模式切换、重置触发 Bing 加载 |
| 数据存储 | 所有读写删操作（IDB + localStorage） |
| 语言系统 | 按钮文字、确认对话框文字 |
| 设置面板 | 图库渲染在设置面板内部，面板打开时触发刷新 |

| 被谁依赖 | 怎么依赖 |
|---------|---------|
| 壁纸系统 | 本地轮换的图片数据由本地图库管理 |
| 设置面板 | 面板打开时调用图库渲染 |
